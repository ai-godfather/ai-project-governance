# Task Files Directory

> This directory contains task files generated by PLANNER (Phase 2).

---

## Structure

```
.ai/tasks/
├── {CATEGORY}/
│   └── {FEATURE_CODE}/
│       ├── TASK_INDEX.md      # Task index for this feature
│       ├── task-001-*.md      # Individual task files
│       ├── task-002-*.md
│       └── ...
└── README.md                   # This file
```

## Categories

Task categories match feature plan categories:

| Category | Description |
|----------|-------------|
| ADMIN | Admin features |
| USER | User-facing features |
| API | API features |
| INFRA | Infrastructure |
| JOB | Background jobs |
| PERF | Performance |
| DEVOPS | DevOps tasks |

## Task File Format

Each task file follows this structure:

```markdown
# Task: {TASK_ID} — {TASK_NAME}

**Feature**: {FEATURE_CODE}  
**Priority**: P0 | P1 | P2  
**Status**: pending | in_progress | completed | blocked  
**Created**: YYYY-MM-DD

---

## Description

[What this task accomplishes]

## Acceptance Criteria

- [ ] Criterion 1
- [ ] Criterion 2

## Files to Modify

- `path/to/file.ext`

## Dependencies

- Task ID (if any)

## Notes

[Additional context]
```

## Workflow

1. **PLANNER Phase 2** generates task files after user approval
2. **AUDITOR** verifies task coverage against plan
3. **IMPLEMENTER** executes tasks and updates status
4. **VALIDATOR** checks implementation

---

**This directory is empty after fresh installation.**  
Task files are generated when you run `@workflow PLANNER` and approve the plan.
